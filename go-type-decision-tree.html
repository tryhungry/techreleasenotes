Answer a few questions to find the right GO type
Question 1 of 3
Is this date already on the meal program's recurring schedule?
Check the Schedule Planner calendar. If the date shows a teal chip, it's on the schedule.
Question 2 of 3
Does this GO need any changes from the program's settings?
Override at the GO level for one-offs: cutoff time, delivery address, subsidy amount, menu selection.
Question 2 of 3
Is this an extra service day for the whole program — same settings, all enrolled users?
If you need specific users only, different settings, or a separate billing path, it should be a Custom Adhoc GO instead.
🔄 No action needed
Recurring GO
The GO auto-generates from the meal program's recurring schedule with standard settings. Nothing to configure — just make sure menus are posted before the cutoff.
Real-world examples
🍱
Weekly team lunchEvery Tuesday the engineering team gets lunch at $18/person. This Tuesday is no different. The GO generates automatically — no action needed from the CSM.
☀️
Daily breakfast programA 200-person office gets breakfast every weekday. Monday through Friday, GOs generate automatically for all enrolled users. The CSM just ensures menus are posted.
✏️ Override at the GO level
Custom GO
The GO auto-generates from the schedule, but you override specific settings for this date only. Open the Schedule Planner, select the date, and edit the fields that need to change. Program defaults stay intact for all other dates.
Real-world examples
👨‍🍳
Guest chef takeoverA special Friday has a guest chef. The CSM increases the subsidy to $30 (from the default $18) and adjusts the cutoff time for that day only. The rest of the schedule is unaffected.
🎉
Holiday party lunchThe December 20th GO needs a higher budget and different delivery address. The CSM overrides those fields for that single GO — all other December Fridays stay at program defaults.
📅 Add the date to the schedule
Added Day GO
Add this date to the meal program's schedule. A GO generates for all enrolled users with the program's standard settings — same as any other service day. If this GO later needs setting changes, you can override them at the GO level after creation. If you need specific users or different settings, create a Custom Adhoc GO instead.
Real-world examples
🏢
Thursday addition to a M–W programThe office normally gets lunch Monday through Wednesday. This week needs Thursday service too. The CSM adds Thursday as an Added Day — all enrolled users see it, same settings as any other service day.
🗓️
Holiday makeup dayMonday was a public holiday so the office was closed. The CSM adds the following Wednesday as a makeup service day. Same settings, all enrolled users — it fits the program's rhythm exactly.
⚡ Create a Custom Adhoc GO
Custom Adhoc GO
Create a Custom Adhoc GO from the Schedule Planner. Unlike an Added Day, you control exactly who gets access — a specific subset of enrolled users. All settings are configurable independently, and it can carry its own billing path. It lives within a meal program but is fully self-contained.
Real-world examples
💼
Executive strategy session8 people from the leadership team are having an offsite lunch. The CSM creates a Custom Adhoc GO scoped to those 8 people, with a $50 subsidy, billed to the leadership cost center. The other 192 enrolled users see nothing.
👋
New hire welcome lunch12 new hires start on Monday. The CSM creates a Custom Adhoc GO scoped to the new hire cohort, with a higher subsidy, billed to HR. Existing employees aren't affected.
🔀
Thursday addition with different settingsThe program is M–W, but this Thursday needs a different dropoff time and higher subsidy. Since it's not the same settings as the program, an Added Day won't work — the CSM creates a Custom Adhoc GO instead.

Quick Reference — All Four GO Types
🔄 Auto-generated

Recurring GO

Generated automatically from the meal program's recurring schedule. Inherits the Client's Default Settings. All enrolled users.

Best for
Any normal scheduled service day — no action needed from the CSM.
✏️ Settings overridden
👥 All Users

Custom GO

A scheduled GO with one or more settings overridden at the GO level for that specific date only. All enrolled users still have access.

Best for
One-off changes within the program — different cutoff, delivery address, or subsidy for a specific date.
📅 Schedule addition
👥 All Users

Added Day GO

A date manually added to the meal program's schedule. Inherits the Client's Default Settings. All enrolled users get access — same as any other service day.

Best for
Makeup days or extra service days for the whole program — same settings, same users. Use Custom Adhoc GO if either of those need to differ.
⚡ Manually created
👤 Subset of Users

Custom Adhoc GO

A one-time GO scoped to a specific subset of enrolled users. All settings are configurable independently. Can carry its own billing path separate from the program.

Best for
Targeted events (exec lunches, new hire cohorts) or any off-schedule GO that needs different users, settings, or billing than the program provides.
💡
The off-schedule shortcut

Adding a date that's not on the schedule? Ask: same settings + all enrolled users?
If yes → Added Day GO. If no (specific users, different settings, or separate billing) → Custom Adhoc GO.
